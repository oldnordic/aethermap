# Plan 12-03: LED Brightness GUI Widget

**Wave:** 2
**Depends on:** 12-01 (IPC handler), 12-02 (Persistence)
**Gap closure:** false
**Autonomous:** true

## Overview

Add an LED brightness slider widget to the GUI. Based on the official Azeron software screenshots, the LED control should be a simple 0-100 slider in a clean interface. The slider should update brightness in real-time as the user drags it.

## Context

The GUI uses the Iced framework. A slider widget already exists and is used for analog calibration. This plan adds a similar slider for LED brightness control.

## Files Modified

- `aethermap/aethermap-gui/src/gui.rs` - Add LED brightness slider to devices tab

## Tasks

<tasks>
1. Read `aethermap/aethermap-gui/src/gui.rs` to understand current GUI structure
2. Add LED brightness state to `GuiState`:
   - `led_brightness_values: HashMap<String, u8>` - Track brightness per device
   - `led_brightness_dragging: Option<String>` - Track which device slider is being dragged
3. Add `Message` variants:
   - `LedBrightnessChanged { device_id: String, brightness: u8 }` - Slider value changed
   - `LedBrightnessDragStarted { device_id: String }` - Slider drag started
   - `LedBrightnessDragEnded { device_id: String }` - Slider drag ended (apply to daemon)
4. Add LED brightness slider to device view (similar to existing analog calibration UI):
   - Section header: "LED Brightness"
   - Slider: 0-100 range with current value display
   - Device name label above slider
5. Implement `update()` handler for LED brightness messages:
   - On change: Update local state only (real-time slider update)
   - On drag end: Send `SetLedBrightness` IPC request to daemon
6. Query current brightness on device load (when device list refreshes)
7. Style the slider to match the existing theme
</tasks>

## Verification Criteria

- [ ] LED brightness slider appears in device view for Azeron Cyborg
- [ ] Slider shows current brightness value
- [ ] Dragging slider updates display in real-time
- [ ] Releasing slider sends IPC request to daemon
- [ ] Brightness change is visible on device
- [ ] Non-LED devices don't show brightness slider

## Must Haves

- 0-100 slider with percentage display
- Real-time visual feedback while dragging
- Actual brightness change applied on drag release (to avoid spamming IPC)
- Current brightness queried on startup
- Only shows for devices with LED capability

## Success Definition

1. User opens GUI, sees Azeron Cyborg device
2. LED Brightness section shows slider at current value (e.g., 75%)
3. User drags slider from 75% to 50%
4. Percentage display updates in real-time: 75% → 70% → 60% → 50%
5. User releases slider
6. IPC request sent to daemon
7. Device LED dims to 50% brightness within 100ms
