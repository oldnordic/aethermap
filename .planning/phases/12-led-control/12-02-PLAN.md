# Plan 12-02: LED Brightness Persistence

**Wave:** 1
**Depends on:** Phase 2 (Per-device profiles)
**Gap closure:** false
**Autonomous:** true

## Overview

Add LED brightness to the profile configuration so that brightness settings persist across profile switches and application restarts. Each profile should store its preferred LED brightness value.

## Context

Profile configurations are stored in YAML format at:
```
~/.config/aethermap/profiles/{device_id}/{profile_name}.yaml
```

The profile structure includes calibration, remappings, and layers. This plan adds an `led_brightness` field to the profile.

## Files Modified

- `aethermap/aethermap-common/src/profile.rs` - Add `led_brightness` field to profile structs
- `aethermap/aethermapd/src/profile_manager.rs` - Load/save LED brightness from profiles
- `aethermap/aethermapd/src/ipc.rs` - Apply brightness when profile switches

## Tasks

<tasks>
1. Read `aethermap/aethermap-common/src/profile.rs` to understand profile structure
2. Add `led_brightness: u8` field to `RemapProfile` struct with default value 100
3. Add LED brightness serialization/deserialization to/from YAML
4. Read `aethermap/aethermapd/src/profile_manager.rs`
5. Modify profile loading to restore LED brightness when profile is activated:
   - After loading profile, get `led_brightness` value
   - Call `led_controller.set_global_brightness(profile.led_brightness).await`
6. Modify profile saving to include current LED brightness
7. Add unit test: profile with LED brightness saves and loads correctly
</tasks>

## Verification Criteria

- [ ] Profile struct includes `led_brightness` field
- [ ] Default brightness is 100% (full brightness)
- [ ] Profile YAML includes `led_brightness` key
- [ ] Switching profiles applies the profile's brightness setting
- [ ] Saving profile captures current brightness value
- [ ] Unit test verifies persistence round-trip

## Must Haves

- Each profile stores its LED brightness preference
- Switching profiles applies the profile's brightness
- Restarting the daemon restores the last active profile's brightness
- Default brightness (100%) is used for profiles without explicit setting

## Success Definition

1. User sets brightness to 50% in "Gaming" profile
2. User switches to "Work" profile (brightness 80%)
3. LED changes to 80% brightness
4. User switches back to "Gaming" profile
5. LED changes to 50% brightness
6. User restarts aethermapd
7. "Gaming" profile's 50% brightness is restored

Example profile YAML:
```yaml
name: Gaming
led_brightness: 50
remappings: [...]
```
