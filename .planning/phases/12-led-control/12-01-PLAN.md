# Plan 12-01: LED Brightness Daemon IPC Handler

**Wave:** 1
**Depends on:** Phase 9 (Azeron device detection)
**Gap closure:** false
**Autonomous:** true

## Overview

Implement the daemon-side IPC handler for LED brightness control. The Azeron Cyborg 2 only supports global brightness control (0-100%), not per-zone RGB color. This plan adds the IPC endpoints that the GUI will use to control brightness.

## Context

The LED controller at `aethermap/aethermapd/src/led_controller.rs` already implements:
- `set_global_brightness(brightness: u8)` - Sets brightness 0-100
- `find_led_interface()` - Finds the correct HID interface
- Keepalive protocol for maintaining connection

The IPC protocol in `aethermap/aethermap-common/src/lib.rs` already defines:
- `Request::SetLedBrightness { device_id, zone, brightness }`
- `Request::GetLedBrightness { device_id, zone }`
- `Response::LedBrightness { device_id, zone, brightness }`

This plan wires up the IPC handlers to call the existing LED controller methods.

## Files Modified

- `aethermap/aethermapd/src/ipc.rs` - Add LED brightness IPC handlers

## Tasks

<tasks>
1. Read `aethermap/aethermapd/src/ipc.rs` to understand current IPC handler structure
2. Read `aethermap/aethermapd/src/led_controller.rs` to understand the LED controller API
3. Add `handle_set_led_brightness()` function in `ipc.rs`:
   - Parse `Request::SetLedBrightness` request
   - Extract device_id and brightness value (0-100)
   - Call `led_controller.set_global_brightness(brightness).await`
   - Return `Response::LedBrightnessSet` with confirmation
4. Add `handle_get_led_brightness()` function in `ipc.rs`:
   - Parse `Request::GetLedBrightness` request
   - Get current brightness from LED controller state
   - Return `Response::LedBrightness` with current value
5. Wire up the handlers in the main IPC match statement
6. Add error handling for:
   - Device not found
   - LED controller not available
   - Invalid brightness values
</tasks>

## Verification Criteria

- [ ] IPC handler accepts `SetLedBrightness` requests
- [ ] IPC handler calls `set_global_brightness()` on LED controller
- [ ] IPC handler returns `LedBrightnessSet` response
- [ ] IPC handler accepts `GetLedBrightness` requests
- [ ] IPC handler returns current brightness value
- [ ] Errors are handled gracefully with appropriate error responses

## Must Haves

- User can set LED brightness via IPC (0-100%)
- User can query current LED brightness via IPC
- Brightness changes take effect immediately on device
- Error responses are clear and actionable

## Success Definition

When a GUI sends a `SetLedBrightness` request with value 50, the daemon:
1. Validates the brightness is in range 0-100
2. Calls the LED controller's `set_global_brightness(50)`
3. Returns `LedBrightnessSet { device_id, zone: None, brightness: 50 }`

The LED on the Azeron Cyborg 2 should reflect the new brightness level.
