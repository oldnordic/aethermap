  <name>Add AutoSwitchRule to config.rs for YAML loading</name>
  <files>aethermap/aethermapd/src/config.rs</files>
  <action>
Add to aethermap/aethermapd/src/config.rs:

1. AutoSwitchRule struct:
   ```rust
   #[derive(Debug, Clone, Serialize, Deserialize)]
   pub struct AutoSwitchRule {
       /// Application identifier to match (e.g., "org.alacritty", "firefox", "*")
       pub app_id: String,

       /// Profile name to activate when this app has focus
       pub profile_name: String,

       /// Device ID to apply profile to (vendor:product format)
       /// If None, applies to all devices
       pub device_id: Option<String>,

       /// Layer ID to activate (0 = base, 1+ = additional layers)
       /// If None, uses profile's default layer
       pub layer_id: Option<usize>,
   }
   ```

2. Add auto_switch_rules field to DeviceConfig:
   ```rust
   pub struct DeviceConfig {
       // ... existing fields ...

       /// Auto-profile switching rules for this device
       #[serde(skip_serializing_if = "Vec::is_empty")]
       #[serde(default)]
       pub auto_switch_rules: Vec<AutoSwitchRule>,
   }
   ```

3. Add global auto_switch_rules to top-level config (for all devices):
   - This allows users to define rules that apply regardless of device

Key design decisions:
- app_id "*" acts as wildcard (matches any app, good for default profile)
- device_id None means rule applies to all devices
- layer_id None uses profile default (typically base layer)
- Rules evaluated in order, first match wins
- YAML format: user-friendly, consistent with existing config

YAML example:
```yaml
auto_switch_rules:
  - app_id: "org.alacritty"
    profile_name: "terminal"
    layer_id: 1
  - app_id: "steam"
    profile_name: "gaming"
  - app_id: "*"
    profile_name: "default"
```

   ```rust
       rules: Vec<AutoSwitchRule>,
       layer_manager: Arc<RwLock<layer_manager::LayerManager>>,
       config_manager: Arc<config::ConfigManager>,
       manual_override: Arc<RwLock<HashMap<String, bool>>>,  // device_id -> override active
   }
   ```

2. Implementation methods:
   - new() - Creates switcher with empty rules
   - load_rules() - Loads rules from all device configs
   - apply_focus_change(app_id, window_title) - Matches rules and switches profiles
   - set_manual_override(device_id) - Marks device as manually overridden
   - clear_manual_override(device_id) - Clears override flag

3. Profile switching logic:
   - Iterate rules in order
   - Match app_id (support "*" wildcard)
   - Check device_id (None = all devices, Some = specific)
   - On match: activate profile and layer
   - Skip if manual override active for device
   - Log profile switches for debugging

4. Integration points:
   - Use LayerManager to switch profiles/activate layers
   - Use ConfigManager to validate profile exists
   - Return Result<(), String> for error handling

Key design decisions:
- Manual overrides persist until next focus change (per CONTEXT decision)
- Rules evaluated in order, first match wins
- No switch when no rule matches (graceful fallback)
- Profile validation before switching (avoid errors)
- Logging at INFO level for profile switches

   ```rust
   Request::FocusChanged { app_id, window_title } => {
       debug!("Focus changed: app_id={}, window_title={:?}", app_id, window_title);

       // Clear manual overrides on focus change
       // Then apply new focus-based profile
       if let Some(switcher) = &self.auto_profile_switcher {
           if let Err(e) = switcher.apply_focus_change(&app_id, &window_title).await {
               warn!("Failed to apply focus change: {}", e);
           }
       }

       let response = Response::FocusChangedAck { app_id: app_id.clone() };
       send_response(&mut stream, &response).await?;
   }
   ```
