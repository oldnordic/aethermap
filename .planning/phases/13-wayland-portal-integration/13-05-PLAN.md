<objective>
Implement global hotkey detection at daemon level

Purpose: Enables manual profile switching via global hotkeys (Ctrl+Alt+Shift+Number).
Since the daemon already grabs input devices, hotkey detection is straightforward:
intercept key events before normal remapping, check for hotkey match, execute action
and consume event if match found. Works on Wayland without compositor support.

Output: GlobalHotkeyManager with configurable key bindings and profile switching
</objective>

   ```rust

       active_modifiers: HashSet<u16>,  // Currently held modifier key codes
       layer_manager: Arc<RwLock<layer_manager::LayerManager>>,
       config_manager: Arc<config::ConfigManager>,
   }
   ```

2. Implementation methods:
   - new() - Creates manager with empty bindings
   - load_bindings() - Loads bindings from all device configs
   - check_key_event(key_code, pressed) -> bool
     * Returns true if hotkey matched and consumed
     * Tracks modifiers (Ctrl, Alt, Shift, Super)
     * Checks for binding match on non-modifier key press
   - trigger_action(binding) - Executes profile switch

3. Modifier tracking:
   - Track LEFT/RIGHT variants separately (KEY_LEFTCTRL=29, KEY_RIGHTCTRL=97)
   - Maintain active_modifiers HashSet for current state
   - Match bindings against active modifier set

4. Key matching logic:
   - Collect all active modifiers when trigger key pressed
   - Normalize modifier names (left/right treated same)
   - Compare against binding modifier list
   - Match trigger key against binding key
   - If match: execute action, return true (consume event)
   - If no match: return false (pass through to remap engine)

5. Action execution:
   - Activate specified profile for device(s)
   - Activate specified layer if provided
   - Set manual override flag (prevents auto-switch until focus change)

Key design decisions:
- Hotkey detection happens BEFORE remap engine (early interception)
- Matching events consumed (not forwarded to applications)
- Non-matching events passed through normally
- Modifiers tracked: Ctrl (29, 97), Alt (56, 100), Shift (42, 54), Super (125, 126)
- Default bindings: Ctrl+Alt+Shift+1-9 for profiles 1-9
- Hotkey press sets manual override (consistent with CONTEXT decision)

2. In event processing loop (before remap_engine.remap()):
   ```rust
   // Check for global hotkeys first
   if let Some(hotkey_manager) = &self.hotkey_manager {
       if event_kind == InputEventKind::Key(key) {
           let key_code: u16 = key.code();
           let pressed = matches!(event.value(), 1);

           if hotkey_manager.check_key_event(key_code, pressed).await {
               // Hotkey matched and consumed, skip remap processing
               continue;
           }
       }
   }

   // Normal remap processing
   self.remap_engine.remap(&mut event, &self.device_id).await?;
   ```
