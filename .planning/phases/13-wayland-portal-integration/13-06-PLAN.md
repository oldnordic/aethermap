2. On GUI startup, create FocusTracker instance
3. Spawn async task for focus monitoring:
   ```rust
   // Start focus tracking
   if let Some(tracker) = FocusTracker::new().await {
       let ipc_client = ipc_client.clone();
       tokio::spawn(async move {
           let mut rx = tracker.start().await?;
           while let Some(event) = rx.recv().await {
               if let Err(e) = ipc_client.send_focus_change(
                   event.app_id,
                   event.window_title
               ).await {
                   eprintln!("Failed to send focus change: {}", e);
               }
           }
           Ok::<(), Box<dyn std::error::Error>>(())
       });
   }
   ```

4. Handle portal permission gracefully:
   - Log warning if portal unavailable
   - Continue GUI operation without focus tracking

Key design decisions:
- Auto-trigger portal request on startup (seamless UX per CONTEXT)
- Focus events sent to daemon via IPC
- Graceful fallback if portal unavailable
- Non-blocking async task for focus monitoring
