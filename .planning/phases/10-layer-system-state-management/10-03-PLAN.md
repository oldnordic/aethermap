Implement toggle mode activation (press on, press off)

Purpose: Enables layers that toggle on/off with each key press. Unlike hold layers,
toggle layers remain active until toggled off again. This is useful for mode switches
(e.g., gaming mode, work mode) that should persist.

Output: Toggle layer activation methods with persistent state tracking
</objective>

1. Add field: `active_toggle_layers: HashSet<usize>` - tracks toggled-on layers

2. Add method: `toggle_layer(&mut self, layer_id: usize) -> bool`
   - If layer_id in active_toggle_layers: remove it, return false (now inactive)
   - If layer_id not in active_toggle_layers: insert it, return true (now active)
   - Returns new state (true=active, false=inactive)

3. Add method: `is_toggle_layer_active(&self, layer_id: usize) -> bool`
   - Returns true if layer_id in active_toggle_layers

4. Update `get_effective_layer(&self) -> usize` (if not done in 10-02)
   - Combine active_hold_layers and active_toggle_layers
   - Return highest layer ID from union, or base_layer if empty

Key design: Toggle layers persist until explicitly toggled off. This differs
from hold layers which deactivate on key release.
