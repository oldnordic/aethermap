<task type="auto">
  <name>Implement layer-aware remap lookup with cascade fallback</name>
  <files>aethermap/aethermapd/src/remap_engine.rs</files>
  <action>
Add to RemapEngine impl:

1. `pub async fn remap_layer_aware(&self, device_id: &str, key_code: Key) -> Option<Key>`
   - Gets effective layer from layer_manager
   - Searches layer_remaps[effective_layer] for key_code
   - If not found, cascades to lower layers (effective-1, effective-2, ... base)
   - If not found in any layer, returns None
   - Implements "lower layers provide defaults" behavior

2. `pub async fn process_event_layer_aware(&self, device_id: &str, key_code: Key, value: i32) -> Option<(Key, i32)>`
   - Same logic as remap_layer_aware but returns (output_key, value) tuple
   - Forwards event value unchanged (preserves press/release/repeat)

Cascade logic:
```rust
let effective_layer = self.layer_manager.read().await.get_effective_layer(device_id);

for layer_id in (0..=effective_layer).rev() {
    if let Some(remaps) = self.layer_remaps.get(layer_id) {
        let remap_table = remaps.read().await;
        if let Some(&output_key) = remap_table.get(&key_code) {
            return Some(output_key);
        }
    }
}
None
```

This searches from highest active layer down to base layer.
