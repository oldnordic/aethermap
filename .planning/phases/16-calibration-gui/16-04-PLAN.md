Update razermapper/razermapperd/src/ipc.rs to handle subscription requests:

1. Add subscriber tracking to DaemonState (around line 200-250):
   ```rust
   /// Analog input subscribers per device
   /// Maps device_id -> list of response senders
   pub analog_subscribers: Arc<RwLock<HashMap<String, Vec<UnboundedSender<Response>>>>>,
   ```
   Note: Use tokio::sync::mpsc::UnboundedSender for fire-and-forget sends.

2. Initialize in DaemonState::new():
   ```rust
   analog_subscribers: Arc::new(RwLock::new(HashMap::new())),
   ```

3. Add handler for SubscribeAnalogInput in handle_request() (around line 1436, after SetAnalogCalibration):
   ```rust
   Request::SubscribeAnalogInput { device_id } => {
       debug!("SubscribeAnalogInput request: device={}", device_id);

       // Create channel for this subscription
       let (tx, _rx) = tokio::sync::mpsc::unbounded_channel();
       let mut subscribers = state.write().await.analog_subscribers;
       subscribers.entry(device_id.clone()).or_default().push(tx);

       Response::AnalogInputSubscribed
   }
   ```

4. Add handler for UnsubscribeAnalogInput:
   ```rust
   Request::UnsubscribeAnalogInput { device_id } => {
       debug!("UnsubscribeAnalogInput request: device={}", device_id);

       let mut subscribers = state.write().await.analog_subscribers;
       subscribers.remove(&device_id);

       Response::Ack
   }
   ```

Note: For simplicity, this implementation removes ALL subscribers for a device. Per-client tracking (requiring client ID) is deferred to Phase 17 if needed. Current approach: one GUI client only.
  </action>
