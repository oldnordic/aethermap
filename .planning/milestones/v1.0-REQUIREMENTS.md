# Requirements: Razermapper - v1.0 Milestone Archive

**Defined:** 2026-02-16
**Shipped:** 2026-02-17
**Core Value:** Users can remap any input key to any other key, with remaps persisting across reboots and switchable at runtime.

## v1 Requirements - COMPLETE

Requirements for initial release. All validated and shipped.

### Core Remapping

- [x] **REMAP-01**: System remaps single input key code to single output key code (e.g., KEY_A → KEY_B) — Delivered via KeyParser (30+ keys) and RemapEngine with HashMap lookup tables (01-01)
- [x] **REMAP-02**: System remaps modifier keys (CapsLock → Ctrl_L, Escape → Ctrl_L, etc.) — Delivered via KeyParser modifier support (01-01)
- [x] **REMAP-03**: Remaps apply only to specified device (per-device profiles) — Delivered via RemapTable type alias and extended YAML config (02-01, 02-02)
- [x] **REMAP-04**: Remaps process with latency under 10ms from input event to output injection — Delivered via O(1) HashMap lookup in RemapEngine (01-01)
- [x] **REMAP-05**: System handles key repeat events (event value 2) correctly for remapped keys — Delivered via event value handling (01-04)
- [x] **REMAP-06**: Remaps prevent stuck keys during device grab (clear existing key states) — Delivered via key state clearing on device grab (01-04)
- [x] **REMAP-07**: System properly handles modifier key state synchronization — Delivered via Drop trait and proper cleanup (01-04)

### Configuration

- [x] **CONFIG-01**: User defines remaps in YAML configuration file with key name syntax — Delivered via KeyParser and YAML config loading (01-02)
- [x] **CONFIG-02**: Daemon auto-loads remap configuration on startup — Delivered via eager config loading at daemon startup (01-03)
- [x] **CONFIG-03**: System reloads configuration without daemon restart (hot-reload) — Delivered via SIGHUP-based hot-reload with validate-then-swap pattern (03-02)
- [x] **CONFIG-04**: User activates/deactivates remap profiles at runtime via IPC — Delivered via IPC protocol extensions for profile operations (02-03, 02-05)
- [x] **CONFIG-05**: Configuration validates before applying (clear error messages for invalid key names) — Delivered via KeyParser validation (01-01)
- [x] **CONFIG-06**: Multiple named profiles supported with easy switching — Delivered via RemapProfile structure and GUI profile switcher (02-06)

### Quality & Reliability

- [x] **QUAL-01**: System processes input events without panics (eliminate unwrap() calls on hot paths) — Delivered via try_read() error handling (01-05)
- [x] **QUAL-02**: Macro engine cleans up completed executions (fix memory leak) — Delivered via cleanup channel (01-05)
- [x] **QUAL-03**: System handles device hotplug (USB unplug/replug) without crashing — Delivered via udev-based device monitoring (03-01)
- [x] **QUAL-04**: Remap functionality covered by unit tests — Delivered via 31 unit tests (486 lines) (04-01)
- [x] **QUAL-05**: Remap functionality covered by integration tests with virtual devices — Delivered via 4 end-to-end tests using evdev::uinput (04-02) and 15 hotplug/reload tests (04-03)
- [x] **QUAL-06**: Daemon releases device grabs gracefully on error/shutdown — Delivered via Drop trait for GrabbedDevice (01-04)

### IPC & Integration

- [x] **IPC-01**: GUI can query active remap configuration — Delivered via GetActiveRemaps IPC request (02-03)
- [x] **IPC-02**: GUI can request remap profile activation — Delivered via ActivateRemapProfile IPC request (02-03)
- [x] **IPC-03**: GUI can request remap profile deactivation — Delivered via DeactivateRemapProfile IPC request (02-03)
- [x] **IPC-04**: IPC protocol supports remap configuration messages — Delivered via RemapProfileInfo and RemapEntry structs (02-03)

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Application-specific remapping | Wayland security model prevents window focus detection |
| X11-specific code paths | Wayland is future; evdev/uinput works on both |
| Network remoting | Security risk; unnecessary attack surface |
| Hardware RGB/LED control | Hardware-specific; not core to remapping value |
| Key sequences (Vim-style) | Complex state machine; defer until v2+ |
| Multi-device macros | Event ordering unreliable across devices |
| Infinite layer nesting | User confusion; cognitive overhead |
| Visual GUI click-to-assign | Significant UI work; defer to v2+ |
